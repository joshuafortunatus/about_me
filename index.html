Platform Migration

Looker → Hex Migration

The problem

  • Poor legacy implementation — Looker had become a "kitchen sink," trying to be everything for everyone for every use case, instead of focusing on who used the data most and what they were actually trying to answer.
  • Data fidelity & definitions — inconsistent metric definitions across dashboards eroded trust in the numbers.
  • Blocker to exploratory analysis — even simple questions were too slow to answer, throttling stakeholders who already had deep domain knowledge from self-serving.

The approach

  • Treated dbt and BI as a single ecosystem, where each layer informs the other, instead of retrofitting BI on top of a legacy model.
  • Centralized metric definitions in one place with a "narrow waist" approach, so every downstream tool inherits the same source of truth.
  • Layered AI-assisted answers on top of that narrow waist — with the right context, an agent can resolve low-to-medium complexity questions for stakeholders who already have deep domain knowledge, closing the gap on simple-but-slow asks.

How

  • Built context guides so similar questions converge on increasingly deterministic answers
  • Version-controlled everything through git
  • Ran hands-on training as adoption rolled out

Outcome

Overwhelming support for Hex during the trial period, with excitement growing as adoption broadened — more trust in the data, and more collaboration across teams.
